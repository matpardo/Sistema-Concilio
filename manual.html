Formularios

Sección que trata de los formularios, sus variables y tipo de variables.

  • buscador

    • Formulario para buscar.
    Div en el que está: id="buscador"
    Variables:

    1. buscar_input

      2. Variable Obligatoria
      String o Int
      1 <= lenth(n) <= 60
      Input de texto
      Representa lo que se va a buscar.

    2. buscar_type

      3. Variable Obligatoria
      Int
      1 <= n <= 4
      Input de select.
      Representa el tipo de elemento que se va a buscar, y significa lo que sigue:
      1 Nº Mesa
      2 RUT Master
      3 Nombre/Nick Master
      4 ID jugador

  • faddmesa

    • Formulario para agregar mesas.
    Div en el que está : id="div_faddmesa" .

    Variables:

    1. addmesa_master

      2. Variable Obligatoria.
      Se obtiene de un select que tendrá la lista de nicks de master.
      Esta lista se obtiene desde la base de datos.

    2. addmesa_number

      3. Variable Obligatoria.
      Int
      0 <= n <= 9999
      La idea es que lo asigne la Base de datos, pero actualmente es un input.

    3. addmesa_ubicacion

      4. Variable Obligatoria
      String.
      0 < length(n) <=50
      Input de texto
      Representa dónde está la mesa.

    4. addmesa_capmax

      5. Variable Obligatoria.
      Int
      0 <= n <= 9999
      Input de texto
      Representa la cantidad máxima de jugadores de la mesa.
      Se usa 0 para representar no límite.

    5. addmesa_capact

      6. Variable Obligatoria
      Se inicia automáticamente
      Int
      0 <= n <= 9999
      Input hidden
      Es la capacidad actual de jugadores registrados en la mesa.
      Se inicia en 0.

    6. addmesa_juego

      7. Variable Obligatoria
      String
      0 < length(n) <= 50
      Input de texto
      Representa el nombre del juego de la mesa.

    7. addmesa_descripcion

      8. Variable opcional (Puede ser NULL)
      String
      0 <= lenght(n) <= 100
      Input de texto
      Representa una descripción breve del juego, (será usado en caso de juegos desconocidos).
      Está pensado para la persona en la mesa de inscripciones.

    8. addmesa_dificultad

      9. Variable Obligatoria
      Int
      1 <= n <= 5
      Input de select.
      Representa el tipo de dificultad de la mesa, y significa lo que sigue:
      1 Novato
      2 Normal
      3 Avanzado
      4 Experto
      5 Cualquiera

    9. addmesa_status

      10. Variable Obligatoria
      Se inicia automáticamente
      Int
      Elemento en {0, 1}
      Input hidden
      Representa la variable de estado de actividad de la mesa: 0 para eliminada y 1 para activa.
      La idea es que cuando se elimine una mesa se cambia este estado, pero no se pierde en la BD.

    10. addmesa_hora

      11. Variable Obligatoria
      Int
      0 <= n <= 23
      Input de texto
      Representa las horas del dia de 0 a 23, cuándo será la mesa.

    11. addmesa_min

      12. Variable Obligatoria
      Int
      0 <= n <= 59
      Input de texto
      Representa los minutos de una hora de 0 a 59, cuándo será la mesa.

    12. addmesa_dia

      13. Variable Obligatoria
      Int
      1 <= n <= 31
      Input de texto
      Representa el día de un mes de 1 a 31, cuándo será la mesa.

    13. addmesa_mes

      14. Variable Obligatoria
      Int
      1 <= n <= 12
      Input de texto
      Representa el mes de un año de 1 a 12, cuándo será la mesa.

    14. addmesa_year

      15. Variable Obligatoria
      Int
      1 <= n <= 9999
      Input de texto
      Representa el año de 1000 a 9999, cuándo será la mesa.

    15. addmesa_info

      16. Variable opcional (puede ser NULL)
      String
      0 <= lenght(n) <= 300
      Input de texto
      Aquí se puede poner cualquier tipo de anotación de la mesa, sirve para quiénes están en la mesa de inscripción.

    16. addmesa_players

      17. Variable Obligatoria
      Se inicia automáticamente
      String
      En la base de datos será un string tal que 1 <= length(n) <= 10000
      Input Hidden
      Representa la lista de jugadores de la mesa.
      Es un estring que empieza con ":" y luego los jugadores se listarán por ejemplo ":jugador1:jugadorr2:jugador3:".
      Se inicia en ":"
      Este diseño es para hacerlo fácil de parsear.

    Variables:

      2. Variable Obligatoria

      1 <= n <=9999
      Input de texto
      Representa número de la mesa a borrar.

    2. delmesa_conf1

      3. Variable Obligatoria
      Int
      1
      Checkbox
      Es la primera confirmación para saber si se está seguro de la operación.

    3. delmesa_conf2

      4. Variable Obligatoria
      Int
      2
      Checkbox

  • faddmaster

    • Formulario para agregar master.
    Div en el que está: id="div_fdelmesa"

    Variables:

    1. addmaster_rut

      2. Variable Obligatoria
      String
      length(n) = 9
      Strings de la forma 12345678K o 01234567K (rut sin puntos ni guiones) que es el rut del master a agregar.

    2. addmaster_name

      3. Su obligatoriedad depende de nick
      String
      0 <= length(n) <= 50
      Input de texto
      Es el nombre del master.

    3. addmaster_nick

      4. Su obligatoriedad depende de nombre
      String
      0 <= length(n) <= 50
      Input de texto
      Es el nick del master.

    4. Oligatoriedad de nick o master

      5. Debe ingresar o nick o nombre, si falta alguno el dato faltante se rellenará internamente con el que si está.

  • faddplayer

    • Formulario para agregar jugador.
    Tener la consideración que los jugadores no se están guardando en la base de Datos, si no que se guardan en el string
    destinado para guardar la lista de jugadores en las mesas, así que si en el identificador del jugador hay ":" hay que limpiarlo (cambiarlo por otro símbolo o algo así).
    Div en el que está: id="div_faddplayer"

    Variables:

    1. addplayer_mesa

      2. Variable Obligatoria
      Int
      1 <= n <=9999
      Input de texto
      Representa número de la mesa a dónde se agregará el jugador.

    2. addplayer_player

      3. Variable Obligatoria
      String
      0 < length(n) <= 20
      Input de texto
      Identificador del jugador a agregar, suele usarse el nº de entrada.
